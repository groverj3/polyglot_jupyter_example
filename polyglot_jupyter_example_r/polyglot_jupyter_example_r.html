Jupyter installed at the global/user level can indeed interact with an IR kernel within an renv.

Document the Environment

Running renv::snapshot() will update the renv.lock. I like to ensure all the packages in the environment are captured with type = "all" to avoid parsing potentially large notebook files and just record everything installed in the environment.

If you've like more intelligent behavior then also saving your notebook as a plain .r script and ignoring all .ipynb files in the project directory with your .renvignore should allow it to record packages and dependencies as intended. However, I opt here for completeness over succinctness.
